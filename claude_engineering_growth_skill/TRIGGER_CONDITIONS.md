# Recommended Trigger Conditions

Use `engineering-growth-mentor` when the user asks about:

- Debugging a bug in a realtime AI system.
- Raspberry Pi 5 deployment.
- Camera pipeline behavior.
- Picamera2/libcamera issues.
- OpenCV image processing issues.
- RGB/BGR or color mismatch.
- YOLO detection accuracy.
- NCNN model inference.
- ROI extraction.
- Classifier input quality.
- Temporal voting or cooldown logic.
- Multithreading, queues, slots, latency, dropped frames, or synchronization.
- Architecture review.
- Code review for a computer vision or embedded AI project.
- Performance bottlenecks.
- Future architecture planning.
- Internship preparation from project experience.
- Learning roadmap or engineering growth.

Do not trigger primarily for:

- Pure code formatting.
- Simple syntax fixes with no system implications.
- General writing tasks unrelated to engineering growth.
- Frontend/UI design tasks unless they relate to this user's engineering portfolio.
